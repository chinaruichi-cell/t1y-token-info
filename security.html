Scanner Notice Explanation

Some wallet or scanner products may flag whitelist, tax, or external-call items by static code pattern. These notices identify that the code contains such mechanisms, but they may not reflect the current on-chain ownership state.

  • Whitelist entries exist for system addresses used by the protocol. New whitelist entries can no longer be added by the project because T1YToken.owner() is the zero address.
  • Sell protection and burn logic is implemented in contract code. Owner-controlled parameter setters are no longer callable after ownership renounce.
  • External calls are used for PancakeSwap Router, WBNB, the reward extension contract, and the fee swapper contract. These calls support liquidity, reward, redemption, and burn mechanics.

  • T1YToken.setFeeSwapper(address)
  • T1YToken.setLiquiditySlippageBps(uint256)
